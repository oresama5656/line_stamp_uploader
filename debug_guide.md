# LINEスタンプ自動化：セレクタ確定ガイド

このガイドは、`line_stamp_uploader` の改修後、最初に ZIP アップロードを成功させるための手順書です。

## 1. 実行前の準備
- `StampHub` の `assets/export/ready/` フォルダに、アップロードしたい ID の ZIP ファイルが配置されていることを確認してください。
- 実行コマンド：
  ```bash
  # 修正したCSVを指定して実行（デバッグモード推奨）
  python main.py d:\StampHub\assets\title_and_info\stamps_metadata_master3.csv --debug
  ```

## 2. ログインと待機
1. ブラウザが立ち上がったら、通常通り LINE にログインしてください。
2. 対象のスタンプ管理画面（「スタンプ情報」のページ）を開きます。
3. ターミナルに戻り、**Enter** を押すと自動入力が開始されます。

## 3. セレクタの確認（現場作業）
メタデータの入力が終わり、画像アップロードのステップに来ると、ターミナルに以下のメッセージが表示され一時停止します：
> `[DEBUG] スタンプ画像タブが開いたことを確認してください。Enter で続行...`

この時、ブラウザ上で以下の点を確認してください：
- **「スタンプ画像」タブ**がクリックされ、開いているか？
- 開いていない場合は、手動でクリックしてからターミナルで Enter を押してください。

## 4. もしエラーが出たら
セレクタが間違っている場合、コンソールに `❌ ZIPアップロード失敗` と表示されます。
その場合は、以下の手順でセレクタを再取得し、`config.py` を更新してください：

1. ブラウザで F12 を押して開発者ツールを開く。
2. 要素選択ボタン（左上の矢印）で、「スタンプ画像」タブや「アップロード」ボタンを選択。
3. 右クリック > **Copy > Copy selector**。
4. `config.py` の以下の箇所に貼り付け：
   - `STAMP_IMAGES_TAB_SELECTOR`
   - `ZIP_UPLOAD_BUTTON_SELECTOR`

## 5. 保存と完了
自動化が正常に動けば、メタデータ保存 → ZIPアップロードまでが流れるように完了します。
2件目以降は、同様の手順で繰り返すか、セレクタが完全に固定されれば `--debug` なしで一括処理可能です。
