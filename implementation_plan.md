# LINEスタンプ登録自動化：シンプル実装ガイド（他AI向け）

「バグの少なさ」「設計のシンプルさ」「改良のしやすさ」を最優先した実装プランです。

## 1. 設計方針：Simple & Sequential
余計な抽象化はせず、上から順に読んで直感的に理解できるコードを目指す。

- **モジュール化しすぎない**: 
  - 1つのスクリプト内で、各工程（ログイン、入力、保存）を明確なセクションに分けて書くこと。これにより、他人が見てどこを直せばいいかすぐわかるようにする。
- **Playwrightの標準機能を活かす**: 
  - 独自のデバッグ機能は作らず、Playwright標準の `page.pause()` や `UI Mode` を活用する指示にする。コード量を減らす。
- **拡張性の確保**: 
  - 入力データ（CSV）の読み込みと、UI操作のロジックのみを分離。将来「画像登録」が増えても、その部分の関数を追加するだけで済むようにする。

## 2. 実装の具体的な指示

### 2.1. 超・デバッグ重視の設定
- **Headed Mode**: 必ず `headless: false` で起動。
- **Slow Motion**: `slow_mo: 1000` (1秒) 程度に設定し、何が起きているか目で追えるようにする。
- **ログイン後の待機**: 
  ```python
  input("ログインが完了し、登録画面を開いたらEnterを押してください...")
  ```
  のような最も原始的かつ確実な方法で自動化をスタートさせる。

### 2.2. バグを未然に防ぐ「操作の基本」
- **オートウェイト**: 要素が見つかるまで待機するPlaywrightの機能を使い、手動で `sleep` を入れないようにする。
- **エラー時の即停止**: 1箇所でも失敗したら、中途半端に処理を続けずその場で停止し、コンソールにエラー原因を出力する。

## 3. 実装のステップ

1. **データ読み込み**: `pandas`等を使わず、Python標準の `csv` モジュールでシンプルに読み込む。
2. **ブラウザ操作**:
   - 言語追加 -> 日本語入力 -> カテゴリ選択、の順でベタ書きに近い形で実装。
   - 複雑なループ制御は避け、1件ごとに「保存して完了」を繰り返す。
3. **拡張用の空欄**:
   - ZIPアップロードを行う箇所に `# [将来の拡張用：ZIP登録処理]` とコメントを残しておく。

## 4. 構成
- `config.py` (オプション): セレクタ（CSS ID等）だけを抜粋したファイル。画面が変わった時にここだけ直せば良いようにする。
- `main.py`: 本体のロジック。
